    • b main

main

  • 在userinit打一个断点
    • b userinit
  • 单步调试或者continue

第一个进程的初始化

  • 在scheduler打一个断点
    • b scheduler
  • 单步调试或者continue
    • s
    • c

kernel/proc.c

initcode是二进制代码的形式，对应initcode.S中的代码，作用是执行系统调用exec("\init", argv)

scheduler

  • 在syscall打一个断点
    • b syscall
  • 单步调试或者continue
    • s
    • c

kernel/pro.c

系统调用

  • 在sys_exec打一个断点
    • b sys_exec
  • 单步调试或者continue
    • s
    • c

kernel/syscall.c

  • 在exec打一个断点
    • b exec
  • 单步调试或者continue
    • s
    • c

kernel/sysfile.c

调试第一个进程

首先退出qemu和gdb，查看init程序的入口地址（Entry point address）

  • 重新启动qemu和gdb，在入口地址加一个断点，b *0x0；
  • 接着，continue，c；
  • 然后加载kernel的调试符号，file user/_init；
  • 查看源代码，layout src；
  • 调试，n。
